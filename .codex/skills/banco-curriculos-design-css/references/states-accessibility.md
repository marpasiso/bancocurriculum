# States and Accessibility Guidelines

- Use semantic colors consistently:
  - success: active subscription, saved actions, completed requests.
  - danger: blocked access, expired subscription, invalid sensitive action.
  - muted: explanations and secondary metadata.
- Ensure color is not the only signal; include clear text.
- Keep contrast high enough for body text and buttons.
- Preserve browser focus outlines or replace them with an obvious accessible focus style.
- Avoid tiny text in forms and admin controls.
- Do not hide critical blocking reasons behind icons alone.
- Keep status messages close to the affected workflow.
